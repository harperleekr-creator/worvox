### 3️⃣ **백엔드 API**

#### 주요 엔드포인트
- `POST /api/rewards/spin` - 랜덤박스 뽑기 (가중 확률 기반)
- `POST /api/rewards/claim` - 상품 수령 정보 제출
- `GET /api/rewards/my-prizes?userId={id}` - 내 당첨 내역 조회

#### 처리 로직
1. 뽑기 시: 레벨 기반 필터링 → 가중 확률 계산 → 랜덤 선택
2. XP 당첨: `xp_transactions` 테이블 자동 기록
3. 실물 상품: `user_prize_wins` 테이블에 당첨 내역 저장
4. 재고 관리: 당첨 시 재고 차감 (XP 제외)

## ✅ 테스트 시나리오

### 1. XP 당첨 테스트
1. 하드 리프레시 (Ctrl+Shift+R)
2. Rewards 탭 이동
3. "열기" 버튼 클릭 → 랜덤박스 클릭
4. 50/100/200 XP 당첨 확인
5. XP가 즉시 반영되는지 확인 (Level Card)
6. "확인" 버튼 → 다시 뽑기 가능한지 확인

### 2. 실물 상품 당첨 테스트
1. 스타벅스/신세계/에어팟/아이패드 당첨 (확률 낮음)
2. 연락처 입력 폼 표시 확인
3. 이름/이메일/전화번호 입력
4. 물리적 상품: 배송지 주소 입력 필드 표시 확인
5. "제출하기" 버튼 클릭
6. 제출 완료 화면 → "내 상품 보기" 버튼 확인

### 3. 내 상품 보기 페이지
1. "내 상품 보기" 클릭
2. 당첨 내역 목록 표시 확인
3. 상태 아이콘: ⏳/📞/✅
4. 당첨일, 운송장 번호 표시 확인
